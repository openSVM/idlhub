Overview

IDLHub is a comprehensive, searchable registry of Interface Definition Language (IDL) files for Solana protocols. It provides both a web interface and programmatic access via REST and MCP APIs.

üìã Comprehensive Coverage

IDLs for 100+ Solana protocols including Jupiter, Orca, Marinade, Drift, Kamino, and many more.

üåê Web Interface

Simple, intuitive web UI for searching, browsing, and exporting protocol IDLs.

üîå REST API

Complete REST API for loading, uploading, and managing IDL files dynamically.

ü§ñ MCP Server

Model Context Protocol server for LLM and editor integration with schema lookup and code generation.

üîç Semantic Search

Powered by Qdrant vector database for intelligent protocol discovery.

üíª Code Generation

Generate TypeScript, Rust, Python, and Anchor TS code from IDL schemas.

Quick Start

Web Interface

Visit idlhub.com to browse and export IDLs using the web interface:

  1. Search for protocols by name or description
  2. Filter by category (DEX, Lending, Derivatives, etc.)
  3. Select protocols you're interested in
  4. Click "Export Selected IDLs" to download them

Programmatic Access

// Load the registry index
const response = await fetch('https://idlhub.com/index.json');
const registry = await response.json();

// Find a specific protocol
const jupiter = registry.protocols.find(p => p.id === 'jupiter');

// Load its IDL
const idlResponse = await fetch(`https://idlhub.com/${jupiter.idlPath}`);
const idl = await idlResponse.json();

MCP Server Installation

# One-line install
curl -fsSL https://idlhub.com/mcp | sh

# Or manual install
git clone https://github.com/openSVM/idlhub.git
cd idlhub
npm install
npm run mcp:start

REST API Server Setup

To run the REST API server locally, follow these steps:

# Clone the repository
git clone https://github.com/openSVM/idlhub.git
cd idlhub

# Install dependencies (REQUIRED)
npm install

# Start the API server
npm run api:start

# The server will be available at http://localhost:3000

Important: You must run npm install before starting the API server. The server requires Express, Axios, CORS, and other dependencies to run.

REST API

The IDLHub REST API provides endpoints for dynamically loading, uploading, and managing Solana program IDL files.

Base URL

For local development: http://localhost:3000

‚ö†Ô∏è Important: The REST API server must be deployed separately. The static site at https://idlhub.com does not include a backend API server. All example commands and "Try It Live" buttons require a running API server at http://localhost:3000. See the REST API Server Setup section above for installation instructions.

Authentication

Currently, the API is open and does not require authentication. For production use, consider adding API key authentication.

POST /api/idl

Create a new IDL record in the registry.

Required Fields:

  • name: Program name (e.g., "jupiter")
  • idl: Complete IDL JSON object with version, name, instructions, accounts, types, etc.
  • address (optional): Solana wallet address for the program
  • category (optional): Program category (defi, dex, lending, etc.)
  • description (optional): Program description

Example Request:

curl -X POST http://localhost:3000/api/idl \
  -H "Content-Type: application/json" \
  -d '{
    "name": "myprogram",
    "address": "MyProg11111111111111111111111111111111111",
    "description": "My custom Solana program",
    "category": "defi",
    "idl": {
      "version": "0.1.0",
      "name": "my_program",
      "instructions": [
        {
          "name": "initialize",
          "accounts": [
            {
              "name": "user",
              "isMut": true,
              "isSigner": true
            }
          ],
          "args": []
        }
      ],
      "accounts": [],
      "types": [],
      "errors": []
    }
  }'

Example Response:

{
  "success": true,
  "message": "IDL created successfully",
  "data": {
    "name": "myprogram",
    "address": "MyProg11111111111111111111111111111111111",
    "version": "0.1.0"
  }
}
GET /api/search

Search for programs using semantic search. This endpoint searches across program names, descriptions, and categories.

Query Parameters:

  • q: Search query (required) - e.g., "dex trading", "lending protocol", "liquid staking"
  • limit: Number of results (default: 10, max: 50)

Example Request:

curl "http://localhost:3000/api/search?q=dex%20trading&limit=5"

Example Response:

{
  "query": "dex trading",
  "total": 5,
  "results": [
    {
      "id": "jupiter",
      "name": "Jupiter",
      "description": "Jupiter Aggregator - The key liquidity aggregator for Solana",
      "category": "dex-aggregator",
      "score": 0.95
    }
  ]
}

MCP API

The Model Context Protocol (MCP) server provides structured, high-performance access to IDL schemas, symbol lookup, code generation, and diagnostics. Perfect for LLM integration and editor tooling.

Features

  • üìã Schema listing & retrieval
  • üîç Symbol lookup (types, instructions, accounts, enums)
  • üíª Code generation (TypeScript, Rust, Python, Anchor TS)
  • ‚úÖ IDL validation and diagnostics
  • üöÄ High performance (p95 < 100ms)
  • üîå Multiple transports (stdio, WebSocket)

MCP Tools

list_schemas

List all available IDL schemas in the registry with optional filtering.

{
  "name": "list_schemas",
  "arguments": {
    "category": "dex",
    "status": "available"
  }
}

get_schema

Retrieve a specific IDL schema by protocol ID.

{
  "name": "get_schema",
  "arguments": {
    "protocol_id": "jupiter"
  }
}

lookup_symbol

Look up types, instructions, accounts, or enums in an IDL.

{
  "name": "lookup_symbol",
  "arguments": {
    "protocol_id": "jupiter",
    "symbol_name": "route",
    "symbol_type": "instruction"
  }
}

generate_code

Generate code from an IDL for a specific target language/framework.

{
  "name": "generate_code",
  "arguments": {
    "protocol_id": "jupiter",
    "target": "typescript"
  }
}

validate_idl

Validate an IDL schema and provide diagnostics.

{
  "name": "validate_idl",
  "arguments": {
    "protocol_id": "jupiter"
  }
}

Custom Client (WebSocket)

import WebSocket from 'ws';

const ws = new WebSocket('ws://localhost:8080');

ws.on('open', () => {
  ws.send(JSON.stringify({
    jsonrpc: '2.0',
    id: 1,
    method: 'tools/call',
    params: {
      name: 'list_schemas',
      arguments: { category: 'dex' }
    }
  }));
});

ws.on('message', (data) => {
  const response = JSON.parse(data);
  console.log(response);
});

Code Examples

JavaScript/TypeScript

// Fetch all IDLs
const response = await fetch('https://idlhub.com/api/all');
const data = await response.json();
console.log(data.idls);

// Get Jupiter IDL by name
const jupiterResponse = await fetch('https://idlhub.com/api/jupiter');
const jupiterData = await jupiterResponse.json();

// Use with Anchor
import { Program, AnchorProvider } from '@coral-xyz/anchor';
const program = new Program(jupiterData.idl, provider);

Python

import requests

# Fetch all IDLs
response = requests.get('https://idlhub.com/api/all')
idls = response.json()['idls']

# Get specific IDL by name
jupiter_response = requests.get('https://idlhub.com/api/jupiter')
jupiter_data = jupiter_response.json()
jupiter_idl = jupiter_data['idl']

# Search for protocols
search_response = requests.get('https://idlhub.com/api/search?q=lending&limit=5')
results = search_response.json()['results']

Rust

use reqwest;
use serde_json::Value;

#[tokio::main]
async fn main() -> Result<(), Box> {
    // Fetch Jupiter IDL by name
    let response = reqwest::get("https://idlhub.com/api/jupiter")
        .await?
        .json::()
        .await?;
    
    println!("Jupiter data: {}", serde_json::to_string_pretty(&response)?);
    Ok(())
}
