Overview

IDLHub is a comprehensive, searchable registry of Interface Definition Language (IDL) files for Solana protocols. It provides both a web interface and programmatic access via REST and MCP APIs.

üìã Comprehensive Coverage

IDLs for 100+ Solana protocols including Jupiter, Orca, Marinade, Drift, Kamino, and many more.

üåê Web Interface

Simple, intuitive web UI for searching, browsing, and exporting protocol IDLs.

üîå REST API

Complete REST API for loading, uploading, and managing IDL files dynamically.

ü§ñ MCP Server

Model Context Protocol server for LLM and editor integration with schema lookup and code generation.

üîç Semantic Search

Powered by Qdrant vector database for intelligent protocol discovery.

üíª Code Generation

Generate TypeScript, Rust, Python, and Anchor TS code from IDL schemas.

Quick Start

Web Interface

Visit idlhub.com to browse and export IDLs using the web interface:

  1. Search for protocols by name or description
  2. Filter by category (DEX, Lending, Derivatives, etc.)
  3. Select protocols you're interested in
  4. Click "Export Selected IDLs" to download them

Programmatic Access

// Load the registry index
const response = await fetch('https://idlhub.com/index.json');
const registry = await response.json();

// Find a specific protocol
const jupiter = registry.protocols.find(p => p.id === 'jupiter');

// Load its IDL
const idlResponse = await fetch(`https://idlhub.com/${jupiter.idlPath}`);
const idl = await idlResponse.json();

REST API

The IDLHub REST API provides endpoints for dynamically loading, uploading, and managing Solana program IDL files.

Authentication

Currently, the API is open and does not require authentication. For production use, consider adding API key authentication.

API Endpoints

GET /health

Check if the API server is running.

{
  "status": "ok",
  "timestamp": "2025-12-10T10:00:00.000Z",
  "version": "1.0.0"
}
GET /api/programs

Get a list of all programs/protocols in the registry.

Query Parameters:

  • category (optional): Filter by category (e.g., dex, lending)
  • status (optional): Filter by status (available or placeholder)
  • search (optional): Search by name or description
curl http://localhost:3000/api/programs?category=dex&status=available
GET /api/programs/:id

Get details for a specific program by ID.

curl http://localhost:3000/api/programs/jupiter
GET /api/programs/:id/idl

Get the full IDL JSON for a specific program.

curl http://localhost:3000/api/programs/jupiter/idl
POST /api/idl/load-from-github

Dynamically load an IDL file from a GitHub repository.

Required Fields:

  • owner: GitHub repository owner
  • repo: GitHub repository name
  • path: Path to IDL file in repository
  • programId: Unique identifier for the program
  • name: Program name
curl -X POST http://localhost:3000/api/idl/load-from-github \
  -H "Content-Type: application/json" \
  -d '{
    "owner": "coral-xyz",
    "repo": "anchor",
    "path": "tests/example.json",
    "programId": "example",
    "name": "Example Program",
    "category": "defi"
  }'
POST /api/idl/upload

Upload an IDL file directly to the registry.

Required Fields:

  • programId: Unique identifier for the program
  • name: Program name
  • idl: Complete IDL JSON object
curl -X POST http://localhost:3000/api/idl/upload \
  -H "Content-Type: application/json" \
  -d '{
    "programId": "myprogram",
    "name": "My Program",
    "category": "defi",
    "idl": {
      "version": "0.1.0",
      "name": "my_program",
      "instructions": []
    }
  }'
GET /api/search

Search for programs using Qdrant semantic search.

Query Parameters:

  • q: Search query (required)
  • limit: Number of results (default: 10)
curl "http://localhost:3000/api/search?q=dex%20trading&limit=5"

MCP API

The Model Context Protocol (MCP) server provides structured, high-performance access to IDL schemas, symbol lookup, code generation, and diagnostics. Perfect for LLM integration and editor tooling.

Features

  • üìã Schema listing & retrieval
  • üîç Symbol lookup (types, instructions, accounts, enums)
  • üíª Code generation (TypeScript, Rust, Python, Anchor TS)
  • ‚úÖ IDL validation and diagnostics
  • üöÄ High performance (p95 < 100ms)
  • üîå Multiple transports (stdio, WebSocket)

MCP Tools

list_schemas

List all available IDL schemas in the registry with optional filtering.

{
  "name": "list_schemas",
  "arguments": {
    "category": "dex",
    "status": "available"
  }
}

get_schema

Retrieve a specific IDL schema by protocol ID.

{
  "name": "get_schema",
  "arguments": {
    "protocol_id": "jupiter"
  }
}

lookup_symbol

Look up types, instructions, accounts, or enums in an IDL.

{
  "name": "lookup_symbol",
  "arguments": {
    "protocol_id": "jupiter",
    "symbol_name": "route",
    "symbol_type": "instruction"
  }
}

generate_code

Generate code from an IDL for a specific target language/framework.

{
  "name": "generate_code",
  "arguments": {
    "protocol_id": "jupiter",
    "target": "typescript"
  }
}

validate_idl

Validate an IDL schema and provide diagnostics.

{
  "name": "validate_idl",
  "arguments": {
    "protocol_id": "jupiter"
  }
}

Custom Client (WebSocket)

import WebSocket from 'ws';

const ws = new WebSocket('ws://localhost:8080');

ws.on('open', () => {
  ws.send(JSON.stringify({
    jsonrpc: '2.0',
    id: 1,
    method: 'tools/call',
    params: {
      name: 'list_schemas',
      arguments: { category: 'dex' }
    }
  }));
});

ws.on('message', (data) => {
  const response = JSON.parse(data);
  console.log(response);
});

Code Examples

JavaScript/TypeScript

// Fetch all available DEX protocols
const response = await fetch('https://idlhub.com/api/programs?category=dex&status=available');
const data = await response.json();
console.log(data.protocols);

// Load Jupiter IDL
const jupiterResponse = await fetch('https://idlhub.com/api/programs/jupiter/idl');
const jupiterIdl = await jupiterResponse.json();

// Use with Anchor
import { Program, AnchorProvider } from '@coral-xyz/anchor';
const program = new Program(jupiterIdl, provider);

Python

import requests

# Fetch all protocols
response = requests.get('https://idlhub.com/api/programs')
protocols = response.json()['protocols']

# Load specific IDL
jupiter_response = requests.get('https://idlhub.com/api/programs/jupiter/idl')
jupiter_idl = jupiter_response.json()

# Search for protocols
search_response = requests.get('https://idlhub.com/api/search?q=lending&limit=5')
results = search_response.json()['results']

Rust

use reqwest;
use serde_json::Value;

#[tokio::main]
async fn main() -> Result<(), Box> {
    // Fetch Jupiter IDL
    let response = reqwest::get("https://idlhub.com/api/programs/jupiter/idl")
        .await?
        .json::()
        .await?;
    
    println!("Jupiter IDL: {}", serde_json::to_string_pretty(&response)?);
    Ok(())
}
